# 安全专家监管规则

1. **数据加密**:
   - 必须使用 **AES-256-GCM** 算法。
   - 密钥（Key）和初始化向量（IV）的管理必须符合安全规范，不能硬编码在代码中。
2. **本地存储**:
   - 敏感配置（如 GitHub Token）必须存储在本地安全位置（如加密的配置文件或系统 Keytar，当前项目要求存本地文件但需注意权限）。
   - 确保 `.zhixia-note` 中的 `config.json` 不会被误传到 GitHub（检查 `.gitignore`）。
3. **IPC 通信**:
   - 渲染进程与主进程通信必须通过 `contextBridge` 暴露有限的 API，禁止开启 `nodeIntegration`。
   - 验证所有文件路径参数，防止路径遍历攻击。
